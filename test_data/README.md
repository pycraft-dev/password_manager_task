# test_data

Синтетические файлы для демонстрации и ручных проверок.

## Файлы

| Файл | Назначение |
|------|------------|
| [sample_records.json](sample_records.json) | Пример **открытого** списка записей (как в памяти после расшифровки). Удобно сверять поля при разработке. Импорт в приложение выполняется из **зашифрованного** JSON через меню «Импорт» (сначала сделайте «Экспорт» в приложении). |
| [invalid_export.json](invalid_export.json) | Некорректный зашифрованный экспорт: ожидается ошибка при «Импорт» (неверная структура / повреждённые данные). |

## Сценарии

- **Успех:** создать хранилище, добавить записи, экспортировать, на чистой копии импортировать с тем же мастер-паролем.
- **Ошибка:** импорт `invalid_export.json` — сообщение об ошибке.
- **Дубликаты:** импорт файла, где названия совпадают с уже существующими — такие записи пропускаются.

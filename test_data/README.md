# test_data

Синтетические файлы для демонстрации и ручных проверок.

## Файлы

| Файл | Назначение |
|------|------------|
| [sample_records.json](sample_records.json) | Пример **открытого** списка записей (как в памяти после расшифровки). Удобно сверять поля при разработке. Импорт в приложение выполняется из **зашифрованного** JSON через меню «Импорт» (сначала сделайте «Экспорт» в приложении). |
| [invalid_export.json](invalid_export.json) | Некорректный зашифрованный экспорт: ожидается ошибка при «Импорт» (неверная структура / повреждённые данные). |
| [chrome_passwords.csv](chrome_passwords.csv) | Пример экспорта Chromium (Chrome / Яндекс.Браузер / Edge / Opera): колонки `name,url,username,password`. Сценарий: **Импорт из браузера** → нужный браузер → Обзор → файл. |
| [edge_passwords.csv](edge_passwords.csv) | То же семейство CSV, демо-строка для Edge. |
| [opera_passwords.csv](opera_passwords.csv) | Формат Chromium, демо для Opera. |
| [yandex_passwords.csv](yandex_passwords.csv) | Формат Chromium, демо для Яндекс.Браузера. |
| [firefox_passwords.csv](firefox_passwords.csv) | Пример Firefox: есть `url`, `username`, `password`. |
| [invalid_export.csv](invalid_export.csv) | Нет нужных колонок — парсер должен выдать ошибку. |

## Сценарии

- **Успех:** создать хранилище, добавить записи, экспортировать, на чистой копии импортировать с тем же мастер-паролем.
- **Ошибка:** импорт `invalid_export.json` — сообщение об ошибке.
- **Дубликаты:** импорт файла, где названия совпадают с уже существующими — такие записи пропускаются.
- **CSV браузера:** открыть хранилище → **Импорт из браузера** → выбрать тип браузера → **Обзор** → предпросмотр → **Импортировать выбранные** или **Импортировать все**; дубликаты по названию (без учёта регистра) пропускаются.
- **ZIP:** можно выбрать архив вместо CSV; если архив с паролем — поле «Пароль ZIP» и кнопка **«Открыть ZIP»** (незашифрованный ZIP — пароль пустой). Берётся первый `.csv` внутри (приоритет имени с подстрокой `password`).
